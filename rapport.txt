Presentation du clien :

Le client est une equipe de devellopement qui travaille sur le projet SXP, qui est une plateforme d'echange peer-to-peer.

Presentation du projet :

Peut on utiliser le pair a pair pour faire un site de pair a pair.
Par exemple : leboncoin.fr -> site pair a pair mais pas implémenté en pair a pair.

SXP permet de :
créer un profil
publier ses offres
faire part de ses demandes
rechercher parmi celles des autres
le tout en peer-to-peer

L'objectif :

L'objectif est de s'assurer que le code actuellement existant ne contient pas de bugs et respecte ses specifications, en integrant des tests unitaires, fonctionels et integrations au processus de compilation.
Cela permet aussi de controller la documentation du projet.

Cahier des charges (besoins client) :

-80% de couverture de code cote serveur (sans Controller)
	Le projet se decoupe en modules (rest, model, crypt, network, protocols, resilience)
	Les modules se decoupent en ensemble de fichiers : les fichiers API(interfaces), Factory(usine), IMPL(implementations)
	En passant par les factory on va pouvoir tester les différentes implémentations des interfaces (tests unitaires).
	Pour mesurer la couverture du code et la présenter au cient on utilisera Jacoco

-Tester l'interface restfull du Controller (Items, Search, Users)
	Il existe un test pour la commande Hash (/command/hash/*), on pourra s'en inspirer pour crer d'autres tests, et cherchera un moyen de les automaztiser.

-Tester la vue
	Utilisation d'une bibliotheque de mocking pour tester la GUI

La periode de devellopement est fixee a TRUC semaines, avec un effectif de 4 personnes.
